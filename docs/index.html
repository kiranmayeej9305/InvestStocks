Configuration

Environment Variables Configuration

All configuration is done through environment variables in the .env.local file.

Groq API Configuration

.env.local 
# Get your free API key from: https://console.groq.com/keys
GROQ_API_KEY=gsk_your_groq_api_key_here
Required: This key is essential for AI chat functionality. The free tier provides generous limits for development and small-scale production.

MongoDB Configuration

.env.local 
# MongoDB Atlas connection string
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/InvestSentry?retryWrites=true&w=majority

Replace username, password, and cluster with your actual MongoDB credentials.

JWT Secret

.env.local 
# Generate with: openssl rand -base64 32
JWT_SECRET=your_super_secret_jwt_key_minimum_32_characters
Security: Use a strong, random string (minimum 32 characters) for production. Never share this key publicly.

Stock Market APIs

.env.local 
# Finnhub API - Real-time stock data
# Free tier: 60 API calls/minute
FINNHUB_API_KEY=your_finnhub_api_key

# Alpha Vantage API - Sentiment & news data
# Free tier: 500 calls/day
ALPHA_VANTAGE_API_KEY=your_alpha_vantage_api_key

Stripe Configuration

.env.local 
# Stripe Keys (use test keys for development)
STRIPE_SECRET_KEY=sk_test_your_stripe_secret_key
STRIPE_PUBLISHABLE_KEY=pk_test_your_stripe_publishable_key

# Product Price IDs (create in Stripe Dashboard)
STRIPE_PRICE_PRO=price_your_pro_plan_id
STRIPE_PRICE_ENTERPRISE=price_your_professional_plan_id

# Webhook Secret (for payment events)
STRIPE_WEBHOOK_SECRET=whsec_your_webhook_secret

Admin Panel (Optional)

.env.local 
# Enable read-only demo mode (for CodeCanyon demos)
# Set to 'true' to enable demo mode, 'false' or omit for production
ADMIN_DEMO_MODE=false
Demo Mode: When set to true, the admin panel operates in read-only mode, perfect for CodeCanyon demos. All viewing features work, but edit/delete/add operations are disabled.
Complete Example: See the included .env.example file for a complete configuration template with all required variables.

Using Feature Flags in Code

Client-side (React components):

typescript 
import { useFeatureFlag } from '@/lib/hooks/use-feature-flag'

function MyComponent() {
  const { enabled, loading } = useFeatureFlag('paper_trading')
  
  if (loading) return 
Loading...

  if (!enabled) return 
Feature disabled

  
  return 
Feature content

}

Server-side (API routes):

typescript 
import { canUseFeatureWithFlags } from '@/lib/plan-limits'

export async function GET(request: NextRequest) {
  const user = await getUserFromRequest(request)
  const canUse = await canUseFeatureWithFlags(user.plan, 'hasPaperTrading')
  
  if (!canUse) {
    return NextResponse.json({ error: 'Feature disabled' }, { status: 403 })
  }
  
  // Feature is enabled, proceed...
}
Note: Feature flags override plan limits. If a feature flag is disabled globally, users won't have access regardless of their plan. Feature flags are checked before plan limits.

  • ‚úì Check API key validity in respective dashboards
  • ‚úì Make sure the file is named .env.local not .env.local.txt

Solution:

  • Verify connection string format is correct
  • Replace <username> and <password> with actual credentials
  • Check Network Access settings allow your IP
  • Ensure database user has proper permissions
  • URL-encode special characters in password

Stripe Payment Fails

Problem: Payments not processing or webhook not working

Solution:

  • Use test card numbers for development: 4242 4242 4242 4242
  • Verify webhook URL is correct and accessible
  • Check webhook signing secret matches
  • Ensure Price IDs are correct in environment variables
  • Test webhooks using Stripe CLI locally

Stock Data Not Loading

Problem: Charts or prices not displaying

Solution:

  • Check Finnhub and Alpha Vantage API keys
  • Verify you haven't exceeded rate limits
  • Check browser console for specific errors
  • Test API keys directly in browser or Postman
  • Ensure stock symbol is valid (U.S. markets)

FAQ

Frequently Asked Questions

Can I use this for commercial projects?

Yes! This is a CodeCanyon regular license product. You can use it for one end product (your website) that end users are not charged for. For multiple projects or charging end users, you need an extended license.

Are all API keys free?

Yes! All required APIs (Groq, Finnhub, Alpha Vantage) offer free tiers that are sufficient for development and small-scale production. Stripe is free to use (they charge transaction fees). Only upgrade when you need higher limits.

Can I customize the design?

Absolutely! The entire UI is built with Tailwind CSS and is fully customizable. Colors, spacing, layouts - everything can be modified to match your brand.

Do I need coding knowledge?

Basic knowledge of JavaScript and React is recommended for customization. However, the installation and configuration can be done by following this documentation step-by-step.

Is it mobile responsive?

Yes! The entire platform is fully responsive and works seamlessly on mobile phones, tablets, and desktops. Special attention has been paid to mobile UX.

Can I add more features?

Yes! The codebase is well-structured and documented, making it easy to add new features. The modular architecture allows for easy extension.

What about SEO?

Built with Next.js 14 App Router, the platform is SEO-friendly with server-side rendering, proper metadata, and semantic HTML. All pages are crawlable and indexable.

Can I change the pricing plans?

Yes! Pricing plans are fully customizable. You can change prices, features, limits, and even add more tiers. All plan logic is centralized in the configuration files.

Is there a demo available?

Yes! A live demo is available on the product page. You can test all features including the AI chat, stock data, portfolio management, and payment flow.

What if I need help?

Support is provided through CodeCanyon's comment system. We typically respond within 24 hours. For urgent issues, you can reach out through the support tab.
