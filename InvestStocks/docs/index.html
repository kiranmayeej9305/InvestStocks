AI-Powered Stock Market Investment Platform with Real-Time Data & Analytics

What is InvestStocks?

InvestStocks is a comprehensive, modern stock market platform built with Next.js 14, React, and TypeScript. It combines AI-powered chat assistance with real-time market data to help investors make informed decisions.

Perfect For: Developers looking to launch their own stock market platform, fintech startups, or investment communities.

Installation

Step-by-Step Installation Guide

1

Extract the Files

Extract the downloaded ZIP file to your desired location.

bash 
unzip investstocks.zip
cd investstocks
2

Install Dependencies

Install all required npm packages using your preferred package manager.

bash 
# Using npm
npm install

# Using pnpm (recommended for faster installation)
pnpm install

# Using yarn
yarn install
Tip: We recommend using pnpm for faster installation and better disk space efficiency.
3

Configure Environment Variables

Copy the example environment file and configure your settings.

bash 
cp .env.example .env.local

Then edit .env.local with your actual values. See the Configuration section for details.

4

Run Development Server

Start the development server to test your installation.

bash 
# Using npm
npm run dev

# Using pnpm
pnpm dev

# Using yarn
yarn dev

Open http://localhost:3000 in your browser to see the application.

5

Set Up Admin Access (Optional)

To access the admin panel, grant admin access to a user account:

bash 
node scripts/make-admin.mjs your-email@example.com

Then log out and log back in with that account. Navigate to /admin to access the admin panel.

See Admin Panel section: For detailed admin panel setup instructions, see the Admin Panel section below.
Success! If you see the InvestStocks landing page, your installation is complete. Continue to the Configuration section to set up your API keys.

Configuration

Environment Variables Configuration

All configuration is done through environment variables in the .env.local file.

Groq API Configuration

.env.local 
# Get your free API key from: https://console.groq.com/keys
GROQ_API_KEY=gsk_your_groq_api_key_here
Required: This key is essential for AI chat functionality. The free tier provides generous limits for development and small-scale production.

MongoDB Configuration

.env.local 
# MongoDB Atlas connection string
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/investstocks?retryWrites=true&w=majority

Replace username, password, and cluster with your actual MongoDB credentials.

JWT Secret

.env.local 
# Generate with: openssl rand -base64 32
JWT_SECRET=your_super_secret_jwt_key_minimum_32_characters
Security: Use a strong, random string (minimum 32 characters) for production. Never share this key publicly.

Stock Market APIs

.env.local 
# Finnhub API - Real-time stock data
# Free tier: 60 API calls/minute
FINNHUB_API_KEY=your_finnhub_api_key

# Alpha Vantage API - Sentiment & news data
# Free tier: 500 calls/day
ALPHA_VANTAGE_API_KEY=your_alpha_vantage_api_key

Stripe Configuration

.env.local 
# Stripe Keys (use test keys for development)
STRIPE_SECRET_KEY=sk_test_your_stripe_secret_key
STRIPE_PUBLISHABLE_KEY=pk_test_your_stripe_publishable_key

# Product Price IDs (create in Stripe Dashboard)
STRIPE_PRICE_PRO=price_your_pro_plan_id
STRIPE_PRICE_ENTERPRISE=price_your_professional_plan_id

# Webhook Secret (for payment events)
STRIPE_WEBHOOK_SECRET=whsec_your_webhook_secret

Admin Panel (Optional)

.env.local 
# Enable read-only demo mode (for CodeCanyon demos)
# Set to 'true' to enable demo mode, 'false' or omit for production
ADMIN_DEMO_MODE=false
Demo Mode: When set to true, the admin panel operates in read-only mode, perfect for CodeCanyon demos. All viewing features work, but edit/delete/add operations are disabled.
Complete Example: See the included .env.example file for a complete configuration template with all required variables.

Database Setup

MongoDB Atlas Setup

1

Create MongoDB Atlas Account

Go to MongoDB Atlas and create a free account.

2

Create a Cluster

Create a new cluster (M0 free tier is sufficient for development).

  • Choose your cloud provider (AWS, Google Cloud, or Azure)
  • Select your preferred region
  • Name your cluster (e.g., "InvestStocks")
3

Create Database User

Navigate to "Database Access" and create a new database user:

  • Choose "Password" authentication
  • Set a username and strong password
  • Grant "Read and Write to any database" permission
4

Configure Network Access

Navigate to "Network Access" and add your IP address:

  • Click "Add IP Address"
  • For development: Select "Allow Access from Anywhere" (0.0.0.0/0)
  • For production: Add specific IP addresses
5

Get Connection String

Click "Connect" on your cluster and choose "Connect your application":

Connection String 
mongodb+srv://<username>:<password>@cluster.mongodb.net/investstocks?retryWrites=true&w=majority

Replace <username> and <password> with your database credentials.

Database Collections

The application will automatically create the following collections:

Collection Purpose Auto-Created
users User accounts and authentication
portfolios User portfolio holdings
watchlists User watchlists
usage API usage tracking

API Keys Setup

Obtaining Required API Keys

Groq API

Purpose: AI chat functionality using LLaMA 3 70B model

Free Tier: Generous limits for development and production

How to Get:

  1. Visit console.groq.com
  2. Sign up for a free account
  3. Navigate to "API Keys"
  4. Click "Create API Key"
  5. Copy the key (starts with gsk_)
The free tier is sufficient for most applications. Groq offers incredibly fast inference speeds.

Finnhub API

Purpose: Real-time stock quotes and market data

Free Tier: 60 API calls per minute

How to Get:

  1. Visit finnhub.io/register
  2. Create a free account
  3. Your API key will be displayed on the dashboard
  4. Copy the key
60 calls/minute is sufficient for most use cases. Consider upgrading for high-traffic applications.

Alpha Vantage API

Purpose: Stock sentiment analysis and news data

Free Tier: 500 API calls per day

How to Get:

  1. Visit alphavantage.co
  2. Enter your email and name
  3. Your API key will be sent via email
  4. Copy the key
500 calls/day is adequate for development. Premium plans available for production.

Admin Panel Setup

Complete Admin Management System

1

Create Your First Admin User

After installation, you need to grant admin access to a user account:

bash

node scripts/make-admin.mjs your-email@example.com

# Or using MongoDB Shell
db.users.updateOne(

Configure Subscription Plans

Go to /admin/plans to set up:

Stripe Setup

Payment Integration Setup

1

Create Stripe Account

Visit Stripe Dashboard and create an account.

2

Get API Keys

Navigate to Developers API keys:

  • Publishable key: Starts with pk_test_ (for test mode)
  • Secret key: Starts with sk_test_ (for test mode)
Use test keys for development. Switch to live keys only when ready for production.
3

Create Products

Navigate to Products and create three pricing plans:

Investor Plan

$19/month

  • Recurring billing: Monthly
  • Description: For active traders

Professional Plan

$49/month

  • Recurring billing: Monthly
  • Description: For professionals

After creating each product, copy the Price ID (starts with price_).

4

Configure Webhook

Navigate to Developers Webhooks Add endpoint:

Webhook URL 
https://yourdomain.com/api/webhooks/stripe

Events to select:

  • checkout.session.completed
  • customer.subscription.updated
  • customer.subscription.deleted
  • invoice.payment_failed

Copy the Webhook signing secret (starts with whsec_).

Testing: Use Stripe's test card numbers to test payments. Card: 4242 4242 4242 4242, Exp: Any future date, CVC: Any 3 digits.

Troubleshooting

Common Issues & Solutions

API Key Errors

Problem: "Missing API key" or "Invalid API key" errors

Solution:

  • Verify all keys are correctly copied in .env.local
  • Ensure no extra spaces or quotes around keys
  • Restart the development server after adding keys
  • Check API key validity in respective dashboards

MongoDB Connection Issues

Problem: Cannot connect to MongoDB Atlas

Solution:

  • Verify connection string format is correct
  • Replace <username> and <password> with actual credentials
  • Check Network Access settings allow your IP
  • Ensure database user has proper permissions
  • URL-encode special characters in password

Stripe Payment Fails

Problem: Payments not processing or webhook not working

Solution:

  • Use test card numbers for development: 4242 4242 4242 4242
  • Verify webhook URL is correct and accessible
  • Check webhook signing secret matches
  • Ensure Price IDs are correct in environment variables
  • Test webhooks using Stripe CLI locally

Stock Data Not Loading

Problem: Charts or prices not displaying

Solution:

  • Check Finnhub and Alpha Vantage API keys
  • Verify you haven't exceeded rate limits
  • Check browser console for specific errors
  • Test API keys directly in browser or Postman
  • Ensure stock symbol is valid (U.S. markets)

Build Errors

Problem: Production build fails

Solution:

bash 
# Clear cache and rebuild
rm -rf .next
rm -rf node_modules
npm install
npm run build
Still Having Issues? Check the browser console and terminal for detailed error messages. Most issues can be resolved by carefully reading the error output.

FAQ

Frequently Asked Questions

Can I use this for commercial projects?

Yes! This is a CodeCanyon regular license product. You can use it for one end product (your website) that end users are not charged for. For multiple projects or charging end users, you need an extended license.

Are all API keys free?

Yes! All required APIs (Groq, Finnhub, Alpha Vantage) offer free tiers that are sufficient for development and small-scale production. Stripe is free to use (they charge transaction fees). Only upgrade when you need higher limits.

Can I customize the design?

Absolutely! The entire UI is built with Tailwind CSS and is fully customizable. Colors, spacing, layouts - everything can be modified to match your brand.

Do I need coding knowledge?

Basic knowledge of JavaScript and React is recommended for customization. However, the installation and configuration can be done by following this documentation step-by-step.

Is it mobile responsive?

Yes! The entire platform is fully responsive and works seamlessly on mobile phones, tablets, and desktops. Special attention has been paid to mobile UX.

Can I add more features?

Yes! The codebase is well-structured and documented, making it easy to add new features. The modular architecture allows for easy extension.

What about SEO?

Built with Next.js 14 App Router, the platform is SEO-friendly with server-side rendering, proper metadata, and semantic HTML. All pages are crawlable and indexable.

Can I change the pricing plans?

Yes! Pricing plans are fully customizable. You can change prices, features, limits, and even add more tiers. All plan logic is centralized in the configuration files.

Is there a demo available?

Yes! A live demo is available on the product page. You can test all features including the AI chat, stock data, portfolio management, and payment flow.

What if I need help?

Support is provided through CodeCanyon's comment system. We typically respond within 24 hours. For urgent issues, you can reach out through the support tab.
